# MatchScore — Output Schema (Bulleted Format)

## Required Output Structure
Return the response in the following sections and order:

### 1) Match Score
- **Match Score:** XX/100
- **Confidence:** Low / Medium / High
- **Summary (1–2 lines):** Short explanation of the score.

### 2) Must-have Requirements Coverage
For each must-have requirement:
- **Requirement:** <text>
- **Status:** Met / Partially Met / Not Evidenced
- **Evidence:** <short CV quote or section reference> (if Met/Partially Met)
- **Notes:** <1 short line>

### 3) Nice-to-have Requirements Coverage
For each nice-to-have requirement:
- **Requirement:** <text>
- **Status:** Met / Partially Met / Not Evidenced
- **Evidence:** <short CV quote or section reference> (if Met/Partially Met)

### 4) Key Strengths (Top 3–5)
- <Strength 1> — Evidence: <CV evidence>
- <Strength 2> — Evidence: <CV evidence>
- <Strength 3> — Evidence: <CV evidence>

### 5) Gaps / Missing Requirements (Top 3–6)
- <Gap 1> — Why it matters: <1 short line>
- <Gap 2> — Why it matters: <1 short line>

### 6) Recommendations (Actionable)
Provide concrete actions:
- **Resume improvements:** what to add / rephrase / quantify
- **Skills to learn:** if relevant
- **Project suggestions:** if relevant

### 7) Red Flags / Unclear Areas (if any)
- <Contradiction or unclear point> — Suggested clarification

## Formatting Rules
- Use bullet points only (no long paragraphs).
- Do not invent experience or tools not in the CV.
- Do not use protected attributes in scoring.
- If evidence is missing, mark as "Not Evidenced" (do not guess).
